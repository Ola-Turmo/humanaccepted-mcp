What is this?

The Model Context Protocol is the standard wiring layer between AI agents and the tools they call — Claude Desktop, Cursor, Claude Code, and dozens of others speak it. humanaccepted-mcp is a passthrough MCP server that wraps every tools/call the agent makes, emitting a pair of signed receipts to the HumanAccepted registry: one before the call (intent + inputs), one after (output + duration + result). Each pair is chained to the previous via chain.prev, and the whole session is tagged with a stable run_id.

All signing happens server-side in the Worker with your tenant's Ed25519 key. The MCP server itself holds no key material — it just authenticates with Bearer sk_… and the Worker does the cryptography. Receipts are content-addressed (the canonical field is the JCS-canonicalized JSON the signature signs over), so anyone with the tenant's public key can verify them offline — no network round-trip needed.

Receipt shape

Receipts follow the open HumanAccepted spec v1.1, which defines three new kinds: tool_call, sub_agent, model_invocation. The MCP server emits tool_call pairs. Minimal example:

{
  "kind": "tool_call",
  "version": 2,
  "tenant": "ten_3K9F2X",
  "id": "rcp_4PZ8QJ2M1X9Y7K0ABCDEF1234",
  "issued_at": "2026-06-28T09:53:11.421Z",
  "agent": {
    "id": "my_agent",
    "type": "claude-sonnet-4.6",
    "run_id": "run_01HF3R8K2Q5P9X7Z4M6V0YJWBN"
  },
  "tool": {
    "name": "get_weather",
    "version": "1.0.0",
    "input": { "city": "Oslo" },
    "output": { "temp_c": 18, "conditions": "cloudy" },
    "duration_ms": 342,
    "result": "ok"
  },
  "chain": {
    "prev": "rcp_4PZ8QJ2M1X9Y7K0ABCDEF1233",
    "run_id": "run_01HF3R8K2Q5P9X7Z4M6V0YJWBN"
  },
  "canonical": "ha:v1:tool_call:ten_3K9F2X:rcp_…:{…}",
  "signature": {
    "alg": "ed25519",
    "key_id": "k_8N4Q",
    "value": "7c3f…"
  }
}

Verify it yourself

The signature is over the canonical field (JCS-canonicalized JSON). Anyone with the tenant's public key can verify offline — no API call needed:

# Python verifier (from humanaccepted-spec/verifier/python)
python3 verifier/python/verify.py \
  --receipt rcp_4PZ8QJ2M1X9Y7K0ABCDEF1234.json \
  --pubkey  tenant_ten_3K9F2X.pub

# TypeScript verifier
npx @humanaccepted/verify \
  --receipt rcp_4PZ8QJ2M1X9Y7K0ABCDEF1234.json \
  --pubkey  tenant_ten_3K9F2X.pub

# Elixir / Go / Rust verifiers also available
ls verifier/

Then walk the chain by following chain.prev until it goes null — that's your full session, in order.

EU AI Act Article 12

Providers of high-risk AI systems must keep automatic, tamper-evident logs of every interaction over the system lifecycle — including tool calls, sub-agent dispatch, and model invocations. humanaccepted-mcp is the drop-in primitive for Article 12: any MCP-compatible agent gets compliant logging with one block of config and zero agent rewrites.
